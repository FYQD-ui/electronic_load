# 通信协议与上位机说明

## 1. 串口参数

- 波特率：115200
- 数据格式：8N1
- 流控：无
- 协议版本：3
- 数据类型：二进制字节流，不再支持 ASCII 文本命令

## 2. 帧格式

```text
55 AA | LEN_L LEN_H | SEQ | CMD | PAYLOAD... | CRC_L CRC_H
```

| 字段 | 长度 | 说明 |
|---|---:|---|
| `55 AA` | 2 | 固定帧头 |
| `LEN` | 2 | `SEQ + CMD + PAYLOAD` 的字节数，小端 |
| `SEQ` | 1 | 请求序号，建议使用 1~255 |
| `CMD` | 1 | 命令码；响应命令码为请求命令码按位或 `0x80` |
| `PAYLOAD` | 0~48 | 命令参数或响应数据 |
| `CRC16` | 2 | CRC16/MODBUS，小端 |

CRC 参数：

- 初值：`0xFFFF`
- 多项式：`0xA001`
- 计算范围：从 `LEN_L` 到 `PAYLOAD` 末尾
- 帧头和 CRC 字段本身不参与计算

示例：设置 1000mA，`SEQ=0x12`：

```text
55 AA 04 00 12 10 E8 03 0B 23
```

## 3. 命令

| 命令 | CMD | 请求 PAYLOAD | 成功响应 PAYLOAD |
|---|---:|---|---|
| 握手 | `0x01` | 无 | `STATUS, PROTOCOL_VERSION` |
| 设备信息 | `0x02` | 无 | 见 4.1 |
| 实时状态 | `0x03` | 无 | 见 4.2 |
| 设置电流 | `0x10` | `current_ma:u16` | `STATUS, current_ma:u16` |
| 控制输出 | `0x11` | `enable:u8`，0/1 | `STATUS, enable:u8` |
| 清除故障 | `0x12` | 无 | `STATUS, fault_bitmap:u32` |

所有多字节整数均使用小端字节序。每个响应 PAYLOAD 的第一个字节都是状态码。

## 4. 响应数据

### 4.1 INFO，CMD=0x82

成功响应 PAYLOAD 固定为25字节：

```text
STATUS:u8
PROTOCOL:u8
FW_MAJOR:u8
FW_MINOR:u8
FW_PATCH:u8
MAX_CURRENT_MA:u16
MAX_VOLTAGE_MV:u16
CONTINUOUS_POWER_MW:u32
ABSOLUTE_POWER_MW:u32
TRIP_TEMPERATURE_MC:i32
CLEAR_TEMPERATURE_MC:i32
```

### 4.2 STATUS，CMD=0x83

成功响应 PAYLOAD 固定为27字节：

```text
STATUS:u8
STATE:u8
OUTPUT_ENABLED:u8
TEMPERATURE_VALID:u8
FAULT_LATCHED:u8
SET_CURRENT_MA:u16
MEASURED_CURRENT_MA:u16
MEASURED_VOLTAGE_MV:u16
MEASURED_POWER_MW:u32
TEMPERATURE_MC:i32
FAULT_BITMAP:u32
RX_OVERFLOW_COUNT:u32
```

运行状态：0=OFF，1=IDLE，2=CC，3=FAULT。

## 5. 状态码

| 值 | 名称 | 含义 |
|---:|---|---|
| `0x00` | OK | 成功 |
| `0x01` | BAD_COMMAND | 未定义命令 |
| `0x02` | BAD_LENGTH | 参数长度或格式错误 |
| `0x03` | RANGE | 参数越界 |
| `0x04` | FAULT_LATCHED | 故障已锁存 |
| `0x05` | TEMP_INVALID | 温度数据无效 |
| `0x06` | FAULT_ACTIVE | 当前仍有故障 |
| `0x07` | IO_ERROR | 硬件操作失败 |

CRC错误、非法LEN和不完整帧不会产生响应。上位机应使用SEQ匹配响应，并在2秒后判定请求超时。

## 6. 故障位

| 位 | 十六进制 | 含义 |
|---|---:|---|
| 0 | `0x01` | 硬件反接/低压 |
| 1 | `0x02` | 硬件过压 |
| 2 | `0x04` | 软件检测过压 |
| 3 | `0x08` | 软件检测过流 |
| 4 | `0x10` | 功率超限 |
| 5 | `0x20` | 过温 |
| 6 | `0x40` | TMP112 通信异常 |
| 7 | `0x80` | MCP4725 DAC 通信异常 |

## 7. PySide6 上位机

```text
doc/electronic_load_gui.py
doc/eload_binary_protocol.py
```

```powershell
pip install PySide6
python .\doc\electronic_load_gui.py
```

上位机自动握手、读取设备信息，并每250ms读取一次状态。通信日志显示完整十六进制收发帧，便于与串口分析工具核对。

断开串口不会自动关闭电子负载输出。需要停止输出时，应先点击“输出关闭”并确认状态更新。
