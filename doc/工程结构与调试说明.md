# 电子负载工程结构与调试说明

## 1. 工程定位

本工程用于 AT32F421C8T7 控制的电子负载，硬件设计目标如下：

- 最大输入电压：12V
- 最大负载电流：3A
- 连续功耗建议上限：25W
- 短时功耗绝对上限：36W

控制方式为 MCU 通过 MCP4725 输出设定电压，LM358 模拟恒流环驱动两只 IRLZ44N MOS 管，R9 50mR 采样电阻形成电流反馈。MCU 负责设定电流、采样电压/电流、读取故障信号、串口命令交互和软件保护。

## 2. 目录结构

```text
electronic_load/
├─ hardware/                 硬件驱动和板级抽象
├─ protocol/                 串口命令协议
├─ project/

```

## 3. 主要模块说明

### 3.1 WorkBench 初始化层

位于 `project/src` 和 `project/inc`，由 AT32 WorkBench 生成或维护：

- `wk_system.c/h`：系统时钟和延时基础。
- `wk_gpio.c/h`：GPIO 初始化。
- `wk_adc.c/h`：ADC1 初始化，目前配置 PA0 和 PA3 双通道序列。
- `wk_dma.c/h`：DMA1 Channel1 初始化，用于 ADC 数据搬运。
- `wk_i2c.c/h`：I2C1 和 I2C2 初始化。
- `wk_usart.c/h`：USART1 115200 初始化。

注意：业务代码尽量不要直接堆在这些 WorkBench 文件里，避免以后重新生成配置时难以维护。

### 3.2 硬件驱动层 `hardware/`

- `hw_adc.c/h`
  - 管理 ADC DMA 双通道缓冲。
  - `PA0`：INA180 电流检测输出。
  - `PA3`：LOAD+ 分压后的电压采样。
  - 提供原始 ADC、mV、mA、负载电压 mV 换算接口。

- `hw_dac_mcp4725.c/h`
  - 通过 I2C1 控制 MCP4725。
  - `MCP4725_ADDR7 = 0x60`，A0 接 GND。
  - 根据目标电流自动换算 DAC raw 值。

- `hw_i2c.c/h`
  - I2C 主机阻塞写接口。
  - 带超时保护，避免 I2C 器件未响应时程序卡死。

- `hw_fault.c/h`
  - 读取 `REV_FAULT` 和 `OV_FAULT`。
  - 故障极性由 `eload_config.h` 中宏控制，便于实测后调整。

- `hw_uart.c/h`
  - USART1 中断接收和阻塞发送封装。
  - 使用128字节环形接收缓冲，并记录溢出次数。

### 3.3 协议层 `protocol/`

- `protocol_cmd.c/h`
  - 二进制串口帧解析、CRC16/MODBUS校验和请求响应。
  - 帧格式：

```text
55 AA | LEN_L LEN_H | SEQ | CMD | PAYLOAD | CRC_L CRC_H
```

  - 支持握手、设备信息、状态读取、电流设置、输出控制和故障清除。
  - 详细字段见 `doc/通信协议与上位机说明.md`。

### 3.4 应用控制层

- `project/inc/eload_config.h`
  - 集中定义硬件参数、保护阈值、换算系数和故障极性。
  - 关键限制：3A、12V、25W 连续、36W 短时。

- `project/src/eload_control.c`
  - 电子负载状态机。
  - 状态包括：`OFF`、`IDLE`、`CC`、`FAULT`。
  - 任何故障、过压、过流、过功率都会立即将 DAC 输出置 0。

- `project/src/main.c`
  - 完成系统初始化。
  - 配置 ADC DMA 真实缓冲区。
  - 初始化控制层和协议层。
  - 主循环每 10ms 执行采样触发、保护判断和串口命令轮询。

## 4. 关键硬件换算

### 4.1 电流测量

硬件参数：

- R9 = 50mR
- INA180A1 增益 = 20V/V

换算关系：

```text
V_ADC_CURRENT = I_LOAD * 0.05R * 20 = I_LOAD * 1.0V/A
```

因此：

```text
1A -> 1.0V
2A -> 2.0V
3A -> 3.0V
```

ADC 以 3.3V 为参考，3A 仍在 ADC 量程内。

### 4.2 负载电压测量

硬件参数：

- R21 = 100K
- R22 = 27K

换算关系：

```text
V_ADC = V_LOAD * 27 / (100 + 27)
V_LOAD = V_ADC * 127 / 27
```

12V 输入时：

```text
V_ADC ≈ 2.55V
```

ADC 量程足够。

### 4.3 电流设定 DAC

硬件参数：

- MCP4725 参考电压约 3.3V
- DAC_OUT 经 R17=100K、R16=4.7K 分压后生成 I_SET
- LM358 恒流环使 `ISENSE ≈ I_SET`

换算关系：

```text
I_SET = DAC_OUT * 4.7K / (100K + 4.7K)
I_LOAD = I_SET / 0.05R
```

3.3V DAC 满量程理论约 2.96A，接近 3A。程序会将目标电流限制在 3000mA，并将 DAC raw 限制在 4095。

## 5. 上板调试流程

### 5.1 上电前检查

1. 先不要连接大功率负载。
2. 检查 12V 输入极性和 3.3V 是否正常。
3. 检查 MOS 管、采样电阻 R9、LOAD+、LOAD-/GND 是否有短路。
4. 检查风扇或散热器是否已安装到位。

### 5.2 首次上电

1. 烧录程序后上电。
2. 启动 `doc/electronic_load_gui.py`，选择串口并连接。
3. 上位机会发送二进制 `PING`、`INFO` 和周期性 `STATUS` 帧。

确认：

- `STATE` 不应为异常状态。
- `REV=0`、`OV=0` 才允许继续。
- 空载电压、电流读数应接近合理范围。

如果 `REV` 或 `OV` 与实际状态相反，先不要带载，修改 `eload_config.h` 中：

```c
#define ELOAD_REV_FAULT_ACTIVE_LEVEL
#define ELOAD_OV_FAULT_ACTIVE_LEVEL
```

### 5.3 小电流闭环验证

建议在上位机中从100mA开始，先设置目标电流，再开启输出。

观察：

- DAC 输出应升高。
- LM358 输出 `GATE_CTRL` 应有变化。
- 采样电阻两端电压约为 `I * 0.05R`。
- 上位机显示的实时电流应接近设定值。

然后逐步测试500mA、1000mA、2000mA和3000mA。

每一步都要确认 MOS 管温度和散热状态。

### 5.4 电压采样验证

给LOAD输入不同电压，对比上位机实时电压与万用表读数。如果误差固定，可在后续版本增加电压校准系数。

### 5.5 故障保护验证

1. 触发过压或比较器故障条件。
2. 程序应进入 `FAULT`，并立即关闭 DAC 输出。
3. 故障解除后，在上位机中点击“清除故障”，再开启输出。

若故障输入仍处于有效状态，`CLR` 后也不应恢复输出。

## 6. 调试注意事项

- 所有输出默认关闭，上电后必须显式执行 `OUT 1`。
- 任何保护触发都会锁存到 `FAULT`，并强制 DAC 输出 0。
- 首次调试不要直接上 3A，应从 100mA 开始逐级增加。
- 25W 连续功耗需要确认散热条件；36W 只能作为短时上限。
- 若串口无输出，先检查 CH340、USART1 PA9/PA10 和波特率。
- 若 I2C DAC 无响应，检查 MCP4725 地址是否为 0x60、SCL/SDA 是否有上拉。
- 若电流读数始终为 0，检查 PA0、INA180 输出、R9 两端压降和 ADC DMA 缓冲。
- 若电压读数异常，检查 R21/R22 分压和 PA3。

## 8. TMP112 温度保护与上位机

当前固件通过 PB14/PB15 软件 I2C 读取 TMP112：

- TMP112 地址为 `0x48`。
- 每 250ms 更新一次温度。
- 80°C 触发关断，降到 70°C 以下才允许清除故障。
- 连续 3 次读取失败会关闭输出并锁存传感器故障。
- LED 在空闲时慢闪、恒流输出时常亮、故障时快闪。

串口协议已升级为版本3二进制帧，不再接收旧ASCII命令。详细协议和PySide6工具使用方法见：

- `doc/通信协议与上位机说明.md`
- `doc/electronic_load_gui.py`
